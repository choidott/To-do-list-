# 2026-07-14 작업 정리

투두 리스트 기능을 완성하고 Vercel에 실제 배포까지 성공한 날.

## 오늘 한 일

### 1. todo 테이블 생성 (책 스키마)
- Supabase SQL Editor에서 `todo` 테이블 생성
  - `id`(text, 앱에서 uuid 생성), `content`, `isCompleted`, `createdAt`, `userId`(user 테이블 참조)
- 이전에 만들었던 임시 `todos` 테이블의 데이터를 이전하고 삭제 → 책과 동일한 구조로 통일
- RLS는 사용하지 않고 애플리케이션 레벨(서버 액션)에서 보안 처리

### 2. 투두 CRUD 완성
| 기능 | 동작 |
|---|---|
| 조회 (Read) | 로그인한 사용자의 할 일만, 최신순 정렬 |
| 입력 (Create) | 엔터로 저장, 성공 시 입력창 비우고 즉시 갱신 |
| 완료 (Update) | 체크박스로 isCompleted 반전, 완료 시 취소선 |
| 삭제 (Delete) | 휴지통 아이콘 + "정말 삭제하시겠습니까?" confirm |

- 모든 쿼리에 `userId` 조건 → 남의 데이터는 조회/수정/삭제 불가

### 3. 폴더(카테고리) 기능
- `folder` 테이블 생성 + `todo.folderId` 컬럼 추가 (NULL 허용, on delete set null)
- 왼쪽 사이드바: 전체 보기 / 폴더 목록 / 새 폴더 추가(+)
- 폴더 클릭 → 해당 폴더의 할 일만 필터링
- 폴더 선택 상태에서 추가하면 자동으로 그 폴더에 저장 (Smart Insert)
- 각 할 일에 폴더 이동 드롭다운 → 기존 할 일도 폴더로 분류 가능
- 폴더 삭제 시 안의 할 일은 지워지지 않고 '전체'로 이동

### 4. 시각적 완성도
- **진행률 표시바**: shadcn/ui Progress, (완료/전체)×100%, 폴더별 계산, 100% 달성 시 "모두 완료했습니다! 🎉"
- **다크 모드**: next-themes, 사이드바 하단 해/달 토글, 색상을 테마 토큰으로 통일
- **로그아웃 버튼**: 사이드바 하단, 클릭 시 세션 종료 후 로그인 페이지로 이동

### 5. Vercel 배포 (트러블슈팅 3연전)
배포 주소: **https://to-do-list-sand-theta.vercel.app**

| # | 에러 | 원인 | 해결 |
|---|---|---|---|
| 1 | No Next.js version detected | GitHub 저장소에 README만 있고 코드가 없었음 | 전체 코드 커밋 후 main 브랜치로 푸시 |
| 2 | Type error: Parameter 'm' implicitly has an 'any' type | @types/pg 미설치 → DB 쿼리 결과가 전부 any | @types/pg 설치 + 쿼리 타입 지정 |
| 3 | 로그인 버튼이 반응 없음 | auth-client에 localhost:3000이 하드코딩되어 빌드에 박힘 | baseURL 제거 → 현재 접속 주소 자동 사용 |

- 서버 쪽 baseURL도 배포 대응: `BETTER_AUTH_URL` → `VERCEL_URL` → `localhost` 순서 폴백
- Google Cloud Console에 배포용 리디렉션 URI 등록: `https://to-do-list-sand-theta.vercel.app/api/auth/callback/google`
- Vercel 환경변수 6개 등록 (환경변수 변경 후엔 반드시 Redeploy!)

## 오늘 배운 것

1. **`.env.local`은 GitHub에 올라가지 않는다** (.gitignore) → 배포 환경변수는 Vercel 대시보드에 따로 등록
2. **`next dev`는 타입 검사를 안 한다** → 로컬에서 잘 돌아도 빌드에서 터질 수 있음. `npx tsc --noEmit`으로 미리 확인
3. **환경변수를 바꾸면 Redeploy 필수** — 기존 배포에는 반영 안 됨
4. **`NEXT_PUBLIC_` 환경변수는 빌드 시점에 코드에 박힌다** → 클라이언트 URL은 하드코딩보다 현재 origin 자동 사용이 안전
5. **한 번 로그인하면 세션 쿠키(7일)로 자동 로그인** — 정상 동작. 시크릿 모드로 새 방문자 화면 확인 가능
