## 오늘 한 일

### 1. 프로젝트 생성
- `C:\Users\chy10\projects\vibe-app`에 Next.js 프로젝트 생성
  - TypeScript, Tailwind CSS, App Router, `src/` 디렉토리 구조
- 패키지 설치: `better-auth`, `@supabase/supabase-js`, `pg`
- shadcn/ui 초기화 (`Button` 컴포넌트 사용)

### 2. 환경변수 설정 (`.env.local`)
- Supabase URL / anon key
- Better Auth secret / URL (`http://localhost:3000`)
- Supabase Postgres 연결 문자열 (`DATABASE_URL`)
- Google OAuth 클라이언트 ID / Secret

### 3. Better Auth 설정
| 파일 | 역할 |
|---|---|
| `src/lib/db.ts` | pg Pool (Supabase 연결, **SSL 필수**) |
| `src/lib/auth.ts` | Better Auth 서버 설정 (구글 소셜 로그인) |
| `src/lib/auth-client.ts` | 클라이언트용 authClient |
| `src/app/api/auth/[...all]/route.ts` | Better Auth API 라우트 |

### 4. 데이터베이스
- Better Auth가 쓰는 테이블 4개가 Supabase에 준비됨:
  - `user` — 회원 기본 정보 (이름, 이메일, 프로필 사진)
  - `session` — 로그인 세션 토큰
  - `account` — 소셜 로그인(구글) 연결 정보
  - `verification` — 이메일 인증 등 일회성 토큰
- 컬럼명은 camelCase라서 SQL에서 반드시 큰따옴표 필요 (예: `"createdAt"`)

### 5. 페이지
| 경로 | 내용 |
|---|---|
| `/` | 홈 — 로그인 페이지로 가는 링크 |
| `/login` | 구글 로그인 버튼 (shadcn/ui outline 버튼 + 구글 G 로고 SVG) |
| `/todo` | 로그인 후 도착하는 대시보드 (자리표시자, 비로그인 시 `/login`으로 리디렉션) |
| `/members` | 회원 명부 — `user` 테이블 조회 (비로그인 시 `/login`으로 리디렉션) |

- 로그인 버튼: `authClient.signIn.social({ provider: "google", callbackURL: "/todo" })`

### 6. 테스트 결과
- `npm run dev` → localhost:3000 정상 구동
- 로그인 버튼 클릭 → `POST /api/auth/sign-in/social` 200 → 구글 로그인 화면으로 정상 리디렉션 확인
- 구글이 앱을 "choidott"로 인식 → OAuth 클라이언트 설정 정상

## 트러블슈팅 기록

1. **`ESSLREQUIRED` 에러** — Supabase pooler는 SSL 연결 필수.
   → `pg` Pool에 `ssl: { rejectUnauthorized: false }` 추가로 해결.
2. **`relation "user" already exists`** — 마이그레이션 CLI 실행 시 테이블이 이미 존재.
   → 이미 스키마가 완성돼 있어서 추가 작업 불필요. SQL은 `create table if not exists`로 작성하면 안전.
